# Plan: Prisma 5→7 Upgrade

## Objective

Upgrade Prisma from 5.14.0 to 7.5.0 with zero regressions. All database operations (CRUD, relations, JSON fields) must continue working correctly.

## must_haves

- [ ] `package.json` has `"prisma": "^7.5.0"` and `"@prisma/client": "^7.5.0"`
- [ ] `npx prisma generate` exits 0
- [ ] `npx prisma db push --accept-data-loss` or `npx prisma migrate dev` works
- [ ] `npm run build` exits 0
- [ ] All database operations work (tested via app)

## Tasks

### Task 1: Upgrade Prisma CLI and Client

<read_first>
- package.json (current Prisma versions: prisma 5.14.0, @prisma/client 5.14.0)
- https://www.prisma.io/docs/orm/more/upgrade-guides/upgrading-versions/upgrading-to-prisma-6 (Prisma 6 breaking changes)
- https://www.prisma.io/docs/orm/more/upgrade-guides/upgrading-versions/upgrading-to-prisma-7 (Prisma 7 breaking changes)
</read_first>

<action>
1. Upgrade both packages together:
   ```bash
   npm install prisma@7.5.0 @prisma/client@7.5.0 --save-dev
   npm install @prisma/client@7.5.0
   ```

   Note: `prisma` is devDependency, `@prisma/client` is runtime dependency.

2. Verify package.json shows:
   - `"@prisma/client": "^7.5.0"` in dependencies
   - `"prisma": "^7.5.0"` in devDependencies

3. Clean install to ensure no version conflicts:
   ```bash
   rm -rf node_modules/.prisma
   npm install
   ```
</action>

<acceptance_criteria>
- `package.json` dependencies contains `"@prisma/client": "^7.5.0"` or `"7.5.0"`
- `package.json` devDependencies contains `"prisma": "^7.5.0"` or `"7.5.0"`
- `npm list prisma` shows version 7.x.x
- `npm list @prisma/client` shows version 7.x.x
</acceptance_criteria>

   ```typescript
   } & typeof global;

   const prisma = globalThis.prismaGlobal ?? prismaClientSingleton()
   ```


5. Common patterns to verify:
   - `findUnique`, `findFirst`, `findMany` — should work
   - `create`, `update`, `delete` — should work
   - `include` for relations — should work
   - `select` for fields — should work
</action>
